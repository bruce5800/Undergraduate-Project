\section{讨论}
\label{sec:discuss}

\subsection{对 AIGC 调度研究的启示}
\label{sec:implications}

我们的实验支持三个超越具体调度器对比的启示。

\textbf{仿真器物理的影响力远超算法精巧度。}本研究中调度器层面最大的效应是关闭连续批处理仿真造成的 $-83\%$ \slo{} 跌幅（表~\ref{tab:ablation}），比任何单一奖励或状态工程选择都大一个数量级。因此 AIGC 调度研究者应把主要精力投入忠实物理建模（连续批处理、KV 本地性、显存带宽下限），而非网络架构或奖励公式。

\textbf{预训练是性价比最高的必备投资。}关闭预训练造成 $38\%$ \slo{} 跌幅（$p<0.001$），但只需 $20$ episode（约 10--15 秒墙钟）。预训练的意义并不在传统意义上的样本效率，而在于\textbf{帮助策略跳出局部最优}---在我们的诊断过程中，早期配置都陷入"always-cloud"吸引子，直到我们将充足预训练与 cloud-overuse 惩罚联合引入。

\textbf{勿过度归因于奖励整形。}一旦连续批处理仿真与预训练就位，没有任何单一 AIGC 奖励项（warm/batch/affinity/cloud-overuse，乃至联合移除）能显著改善 \slo{}。我们推测：先前声称某具体奖励项贡献的 AIGC 调度工作，可能忽略了充足表达力的 PPO 主干内部，奖励信号的\textbf{相互可替换性}。今后只展示单项奖励正向效果的工作\emph{应当怀疑}：除非给出表~\ref{tab:ablation} 这种联合消融。

总体上，这三条启示提示 AIGC 调度研究议程应当从"造更精巧的奖励"转向\textbf{"造更忠实的仿真器与训练流程"}。

\subsection{局限与有效性威胁}
\label{sec:limitations}

\textbf{仿真保真度。}我们将连续批处理建模为\emph{准入式}批处理而非 vLLM 的迭代级动态再批处理，在某些工况下可能低估真实连续批处理的收益 10--20\%；显存带宽下限也用了与 batch size 无关的常数（实际 HBM 争用按 batch 子线性扩展）。定量数字（如 \texttt{no\_batching} 的 $-83\%$）应理解为我们仿真器的行为，而非 vLLM 的精确预测。

\textbf{工作负载与硬件覆盖。}我们的负载基于对齐 Azure 2023 公开统计的对数正态长度合成数据，未回放专有生产 trace，也未覆盖交互式应用常见的突发到达模式；硬件涵盖 A100/T4/Jetson 层级，但未含 H100、MI300X 或专有推理加速器（如 Groq）。所建模的 AIGC 物理特性架构无关、应可推广，但参数常数需对新硬件重新标定。

\textbf{统计功效与算法范围。}$N=30$ 检测中等以上效应（Cohen's $d \ge 0.5$）有效，但无法分辨小效应；消融的零结果意味着"未能检测到大于 5\% 的 \slo{} 差异"，而非"不存在差异"。我们测试了 PPO、A3C-R2N2 与一个 GNN 变体，三者均呈现类似的 AIGC 可替换性；但 offline RL / 基于模型的 RL / 来自启发式 oracle 的模仿学习，可能呈现不同响应。

\subsection{部署实践指导}
\label{sec:guidance}

实践者部署 AIGC 调度器时可参考以下四点。其一，\textbf{容量规划比调度器选择更重要}：\S\ref{sec:workload} 的 large-heavy 实验显示，当 70B 请求溢出唯一云端服务器时，所有调度器 \slo{} 都被压到约 10\%，远不及"加一台云端 GPU"带来的 3$\times$ \slo{} 提升---\emph{先备算力，再优调度}。其二，\textbf{中等负载工况（$\lambda \le 2$ + 多样化模型混合）下简单负载均衡器足矣}：LeastLoaded 与 ShortestQueue 在 \slo{} 上达到 RL 的 78\%（图~\ref{fig:pareto}）却零训练成本，更适合小规模或缺 ML 工程能力的部署。其三，\textbf{当能效或尾延迟重要时再上 RL}：本文 RL 在 30 种配置下保持 5--11\% 的每 token 能耗优势，在超大规模部署中每年节省可达数百万美元，且尾 TTFT（P95 25.2~s vs.\ 基线 28--41~s）对交互式应用有显著意义。其四，\textbf{对奖励工程的声明保持怀疑}：审视新提出的 AIGC 奖励组件时，应问：是否在含有预训练基线之上做了联合消融？没有该控制，归因于具体奖励的增益可能仅反映通用 RL 主干能力的提升，与所声称机制无关。
